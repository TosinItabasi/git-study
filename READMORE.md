# Gitの勉強
- git add コマンドで、リポジトリに変更情報を追加する
    - このことをステージングと言う
- git commit コマンドで、リポジトリのインデックスに追加された変更情報にコメントを付けてコミットできる
- git push コマンドで、ローカルのコミットをリモートのリポジトリに反映させることができる

## clone (クローン)
GitHub上のForkと同じ
Linux 内にも GitHub のようなリポジトリをつくること

## pull （プル）
GitHub 上でコミットがあった場合などにそれを取得すること
`git pull origin gh-pages`
などのコメントでプルすることができる

### origin
`origin`はクローンの元になったリモートレポジトリのデフォルト名

※ここでは…
リモートレポジトリ：github
ローカルレポジトリ：git

## レポジトリをローカルにつくる
`git init`
削除したい場合は.git ディレクトリを丸ごと削除すればOK

## ファイルをレポジトリに追加したい
`git add hogehoge(READMORE.md)`
このコマンドで「ファイルをリポジトリに追加したい」という変更情報を登録
（まだ登録-コミット-はできていない）
ステージング（コミットの前）の状態になっている

### ステージングの状態を確認したい
`git status`
